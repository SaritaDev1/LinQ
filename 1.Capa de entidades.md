# Capa de entidades

La capa de entidades es la parte del sistema donde se crean las clases que representan la información de la base de datos.

Para entenderlo de forma sencilla, una entidad es como una representación en C# de una tabla de la base de datos. Es decir, si en la base de datos existe una tabla, en la capa de entidades se puede crear una clase que represente esa información.

Cada columna de la tabla se convierte en una propiedad dentro de la clase. De esta manera, el sistema puede trabajar con objetos en lugar de manejar los datos de forma suelta.

En este manual se toma como ejemplo la base de datos `Northwind`, porque contiene varias tablas relacionadas, como clientes, empleados, pedidos, productos y categorías.

Por ejemplo, en `Northwind` existe la tabla Customers, que almacena información de los clientes. Esta tabla puede representarse en C# con una clase llamada Customer.

```text
public class Customer
{
    public string CustomerID { get; set; }
    public string CompanyName { get; set; }
    public string ContactName { get; set; }
    public string City { get; set; }
    public string Country { get; set; }
}
```

En este ejemplo, la clase Customer representa la tabla Customers de Northwind.

Las propiedades CustomerID, CompanyName, ContactName, City y Country representan algunas columnas de esa tabla.

También se pueden crear otras entidades según las tablas que se necesiten usar, por ejemplo:

```text
Customers     → Customer
Products      → Product
Orders        → Order
Employees     → Employee
Categories    → Category
Order Details → OrderDetail
```

No siempre es necesario crear una clase con todas las columnas de una tabla. En algunos casos se colocan solo los campos que el sistema necesita utilizar.

La capa de entidades sirve para representar las tablas de la base de datos mediante clases de C#. En este caso, `Northwind` se usa como referencia para entender cómo una tabla puede convertirse en una clase dentro del sistema.
