# Capa de negocio

La capa de negocio es la parte del sistema donde se colocan las reglas y validaciones.

Esta capa no muestra pantallas ni se conecta directamente a la base de datos. Su trabajo es revisar que la información sea correcta antes de enviarla a otra capa.

En este manual se toma como ejemplo la base de datos `Northwind`, donde existen datos de clientes, productos, pedidos y empleados.

LINQ también puede participa en la capa de negocio, pero de una forma diferente a la capa de acceso a datos.

En la capa de acceso a datos, LINQ se usa para consultar la base de datos. En cambio, en la capa de negocio, LINQ se puede usar para filtrar o revisar información que ya fue obtenida.

Por ejemplo, la capa de acceso a datos puede traer todos los productos de `Northwind`, y la capa de negocio puede usar LINQ para mostrar solo los productos disponibles.

```csharp
public List<Product> ObtenerProductosDisponibles()
{
    var productos = datosProducto.ListarProductos();

    var disponibles = from p in productos
                      where p.Discontinued == false
                      select p;

    return disponibles.ToList();
}
```

En este ejemplo, `datosProducto.ListarProductos()` trae los productos desde la capa de acceso a datos.

Luego, LINQ usa `where p.Discontinued == false` para quedarse solo con los productos que no están descontinuados.

Por eso LINQ puede participar en la capa de negocio: porque ayuda a aplicar reglas, filtros o validaciones antes de enviar la información a la capa de presentación.
