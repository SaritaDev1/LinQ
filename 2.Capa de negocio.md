# Capa de negocio

La capa de negocio es la parte del sistema donde se colocan las reglas y validaciones.

Esta capa no muestra pantallas ni se conecta directamente a la base de datos. Su trabajo es revisar que la información sea correcta antes de continuar con el proceso.

Tomando como ejemplo `Northwind`, si el usuario desea buscar clientes por país, primero escribe el país en la capa de presentación. Luego, ese dato llega a la capa de negocio.

En la capa de negocio se puede validar que el país no esté vacío antes de llamar a la capa de acceso a datos.

```csharp
public List<Customer> ObtenerClientesPorPais(string pais)
{
    if (pais == "")
    {
        return new List<Customer>();
    }

    var clientes = datosCliente.ObtenerClientesPorPais(pais);

    return clientes;
}
```

En este ejemplo, el método `ObtenerClientesPorPais(pais)` recibe el país que viene desde la capa de presentación.

Primero se revisa si `pais` está vacío. Si no se escribió nada, se devuelve una lista vacía. Esto evita hacer una consulta innecesaria a la base de datos.

Si el país sí tiene un valor, la capa de negocio llama al método `datosCliente.ObtenerClientesPorPais(pais)`, que pertenece a la capa de acceso a datos.

En la capa de acceso a datos es donde se usa LINQ para consultar la tabla `Customers` de `Northwind`.

La variable `clientes` recibe una lista de objetos `Customer`. Cada objeto representa un cliente que cumple con la búsqueda realizada.

En este caso, LINQ no se escribe directamente en la capa de negocio, porque la consulta principal está en la capa de acceso a datos. Pero la capa de negocio sí participa en el proceso, porque valida el dato antes de permitir que se realice la consulta.

La capa de negocio también puede trabajar con otros datos, no solo con clientes. Por ejemplo, puede revisar productos, pedidos o empleados.

Si se desea obtener productos disponibles, la capa de negocio puede llamar a la capa de acceso a datos para traer los productos y luego revisar cuáles se pueden mostrar.

```csharp
public List<Product> ObtenerProductosDisponibles()
{
    var productos = datosProducto.ListarProductos();

    var disponibles = from p in productos
                      where p.Discontinued == false
                      select p;

    return disponibles.ToList();
}
```

En este ejemplo, `datosProducto.ListarProductos()` llama a la capa de acceso a datos y obtiene los productos de `Northwind`.

Después, LINQ se usa en la capa de negocio para filtrar esa lista y quedarse solo con los productos que no están descontinuados.

Aquí LINQ participa de una forma diferente. No está consultando directamente la base de datos, sino trabajando con una lista de objetos `Product` que ya fue obtenida.

Esto significa que LINQ puede participar en la capa de negocio cuando se necesita filtrar, ordenar o revisar datos antes de enviarlos a la capa de presentación.

También se pueden crear otros métodos en la capa de negocio, por ejemplo:

```text
ObtenerClientesPorPais()
ObtenerProductosDisponibles()
BuscarProductoPorId()
ValidarProducto()
ListarPedidos()
```

La diferencia es que la capa de negocio no debería encargarse directamente de consultar la base de datos. Para eso existe la capa de acceso a datos.

La capa de negocio se encarga de revisar reglas, validar datos y preparar la información.

LINQ participa principalmente en la capa de acceso a datos, porque ahí se consulta la base de datos. También puede participar en la capa de negocio, pero cuando se necesita trabajar con información que ya fue obtenida. Por eso, la capa de negocio ayuda a controlar el proceso antes de enviar los resultados a la capa de presentación.
