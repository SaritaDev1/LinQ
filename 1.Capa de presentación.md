# Capa de presentación

La capa de presentación es la parte del sistema que ve y utiliza el usuario.

En esta capa se encuentran las pantallas, formularios, botones y tablas donde se ingresa o muestra información.

Esta capa no debe conectarse directamente a la base de datos ni escribir consultas LINQ directamente. Su trabajo es recibir datos del usuario y enviarlos a la capa de negocio.

Tomando como ejemplo `Northwind`, si el usuario desea buscar clientes por país, la presentación recibe el país escrito y llama a la capa de negocio.

```csharp
private void btnBuscar_Click(object sender, EventArgs e)
{
    string pais = txtPais.Text;

    var clientes = negocioCliente.ObtenerClientesPorPais(pais);

    dgvClientes.DataSource = clientes;
}
```

En este ejemplo, la capa de presentación no usa LINQ directamente. Solo llama a `negocioCliente.ObtenerClientesPorPais(pais)` y muestra los resultados.

LINQ participa después, en la capa de negocio o en la capa de acceso a datos, donde se filtra o consulta la información de `Northwind`.

En resumen, la capa de presentación sirve para interactuar con el usuario, pero no debe consultar directamente la base de datos.
