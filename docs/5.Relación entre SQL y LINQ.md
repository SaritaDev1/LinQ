# Relación entre SQL y LINQ

SQL y LINQ sirven para consultar información, pero se escriben de diferente manera.

SQL trabaja directamente con tablas y columnas de la base de datos.

LINQ trabaja dentro de C#, usando clases, objetos y propiedades.

Por ejemplo, en la base de datos `Northwind` existe la tabla `Customers`, que contiene información de los clientes.

Si se desea buscar los clientes que pertenecen a `Germany`, en SQL se puede escribir así:

```sql
SELECT *
FROM Customers
WHERE Country = 'Germany';
```

La misma consulta en LINQ se puede escribir así:

```csharp
var consulta = from c in db.Customers
               where c.Country == "Germany"
               select c;
```

Las dos consultas buscan clientes de `Germany`, pero funcionan de forma diferente.

En SQL se trabaja directamente con la tabla `Customers` y con la columna `Country`.

En LINQ, `db.Customers` representa la tabla `Customers`, y `c.Country` representa la propiedad `Country` de la entidad `Customer`.

La parte `where` sirve para filtrar la información. Esto significa que no se muestran todos los clientes, sino solo los que cumplen la condición.

La parte `select c` indica que se selecciona el cliente completo.

En una arquitectura de 4 capas, LINQ participa principalmente en la capa de acceso a datos, porque ahí se realizan las consultas a la base de datos.

También puede participar en la capa de negocio cuando se necesita filtrar, ordenar o revisar información que ya fue obtenida.

La diferencia principal es que SQL se escribe directamente para la base de datos, mientras que LINQ se escribe dentro del código de C#.
