# Capa de negocio

La capa de negocio es la parte del sistema donde se colocan las reglas y validaciones.

Esta capa no se encarga directamente de mostrar pantallas ni de conectarse a la base de datos. Su trabajo principal es revisar que la información sea correcta antes de enviarla o recibirla de otras capas.

Por ejemplo, en la capa de negocio se puede validar que un campo no esté vacío, que un valor sea mayor que cero, que un cliente esté activo o que un préstamo cumpla ciertas condiciones antes de ser registrado.

Esta capa trabaja como intermediaria entre la capa de presentación y la capa de acceso a datos.

La capa de presentación envía la información que ingresa el usuario. Luego, la capa de negocio revisa esa información. Si todo está correcto, se comunica con la capa de acceso a datos para consultar o guardar información en la base de datos.

Un ejemplo de validación en la capa de negocio sería:

```csharp
public bool RegistrarCliente(Cliente cliente)
{
    if (cliente.Nombre == "")
    {
        return false;
    }

    if (cliente.Apellido == "")
    {
        return false;
    }

    datosCliente.GuardarCliente(cliente);
    return true;
}
```

En este ejemplo, primero se revisa que el nombre y el apellido no estén vacíos. Si los datos son correctos, recién se llama a la capa de acceso a datos para guardar el cliente.

La capa de negocio sirve para controlar las reglas del sistema. Ayuda a que la información no se guarde o se muestre sin antes ser revisada.
