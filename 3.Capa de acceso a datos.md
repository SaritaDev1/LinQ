# Capa de acceso a datos

La capa de acceso a datos es la parte del sistema que se encarga de comunicarse con la base de datos.

En esta capa es donde LINQ participa principalmente, porque aquí se realizan las consultas para obtener información. LINQ permite escribir consultas dentro de C# para buscar, listar o filtrar datos de la base de datos.

En este manual se toma como ejemplo la base de datos `Northwind`, donde existen tablas como `Customers`, `Products`, `Orders`, `Employees` y `Categories`.

La capa de acceso a datos no se encarga de mostrar pantallas ni de aplicar reglas del sistema. Su trabajo principal es consultar la base de datos y devolver la información a la capa de negocio.

Por ejemplo, si el usuario desea buscar clientes por país, la capa de presentación recibe el país, la capa de negocio valida el dato y luego la capa de acceso a datos realiza la consulta usando LINQ.

```csharp
public List<Customer> ObtenerClientesPorPais(string pais)
{
    var consulta = from c in db.Customers
                   where c.Country == pais
                   select c;

    return consulta.ToList();
}
```

En este ejemplo, `db.Customers` representa la tabla `Customers` de `Northwind`.

La parte `from c in db.Customers` indica que la consulta va a trabajar con los clientes.

La parte `where c.Country == pais` sirve para filtrar la información. Esto significa que no se van a traer todos los clientes, sino solamente los clientes cuyo país sea igual al valor recibido.

La parte `select c` indica que se selecciona el cliente completo.

Finalmente, `ToList()` convierte el resultado en una lista de objetos `Customer`, para que esa información pueda pasar a la capa de negocio.

También se pueden crear otros métodos en la capa de acceso a datos, dependiendo de la información que se necesite consultar.

Por ejemplo, si se desea listar todos los productos de `Northwind`, se puede crear un método como este:

```csharp
public List<Product> ListarProductos()
{
    var consulta = from p in db.Products
                   select p;

    return consulta.ToList();
}
```

En este caso, LINQ consulta la tabla `Products` y devuelve una lista de objetos `Product`.

También se podría buscar un producto específico por su código:

```csharp
public Product BuscarProductoPorId(int id)
{
    var consulta = from p in db.Products
                   where p.ProductID == id
                   select p;

    return consulta.FirstOrDefault();
}
```

En este ejemplo, `where p.ProductID == id` filtra el producto según su código.

Se usa `FirstOrDefault()` porque se espera obtener un solo producto o ningún resultado.

Esto demuestra que LINQ puede devolver diferentes tipos de resultados. Si se necesitan varios registros, se puede usar `ToList()`. Si se necesita un solo registro, se puede usar `FirstOrDefault()`.

Algunos métodos que podrían existir en la capa de acceso a datos son:

```text
ListarClientes()
ObtenerClientesPorPais()
ListarProductos()
BuscarProductoPorId()
ListarPedidos()
ListarEmpleados()
```

Por eso, LINQ participa en la capa de acceso a datos porque esta capa es la que consulta directamente la base de datos.

La capa de acceso a datos usa LINQ para obtener información de `Northwind` de forma ordenada. Gracias a esta capa, la presentación no se conecta directamente a la base de datos y la capa de negocio recibe la información que necesita para aplicar reglas o validaciones.
